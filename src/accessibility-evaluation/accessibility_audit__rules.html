Accessibility guidelines for developers

Number Guideline Priority
Form elements must have an accessible name. Critical
Form elements in the invalid state must have a HTML attribute `aria-invalid`. Critical
The web page must have a `header`, `main` and `footer` elements. Critical
A HTML element `a` must not behave as a button. Critical
A HTML element `button` must not behave as a link. Critical
HTML tables must be well formed. Critical
The user must be able to use common keyboard shortcuts to go back to the previous page, or to move forward to the next page. Critical
The user must be able to use common keyboard shortcuts to scroll down or up the web page. High
A HTML element `iframe` must always have a `title` attribute. High
Form elements that are required for form submission must have a HTML attribute `aria-required`. High
There must be no interactive elements as descendants of other interactive elements. High
The CSS property `box-shadow` should be avoided. High
The HTML `role` attribute must always have a valid value. High
There must be no boxes with a fixed height in CSS. High
A group of radio button with a HTML attribute `name` of equal value must be descendants of a HTML element `fieldset`. High
A group of checkboxes with a HTML attribute `name` of equal value must be descendants of a HTML element `fieldset`. High
The HTML attribute `maxLengh` should be avoided. High
Form elements should have a HTML attribute `autocomplete` if appropriate. High
Each occurrence of the HTML attribute `id` must have a unique value. High
The HTML attribute `role` with a value of `group` must be used to define the start and the end of content that shows and hides. High
The HTML attribute `aria-current` must be used to distinguish the active item in a group of items. High
Heading levels should only increase by one. High
A HTML element `table` must always have an accessible name. Medium
A HTML element `img` must always have an `alt` attribute. Medium
A HTML element `button` or `link` must always have text content. Medium
The HTML element `html` must have a `lang` attribute with a valid value. Medium
There must not be any character key shortcuts. Medium
There must at least and at most one HTML element `h1` on a web page. Medium
Labels for form elements must be unique. Medium
The HTML attribute `tabindex` with a value greater than `0` must be avoided. Medium
The `order` CSS property with a value other than `0` should be avoided in CSS. Medium
Absolute, fixed, relative and sticky positioning should be avoided in CSS. Medium
The text alternative of images must be must be clearly visible and legible when the images fail to load on the web page. Low
The HTML attribute `hidden` must be used instead of the CSS property `display` with a value of `none`. Low
The content of the HTML element `title` must be same as the content of the HTML `h1` element. Low
The CSS property `animation` should be avoided unless the web page can honor the user's request to disable animations. Low
The web page must have sufficient colour contrast. Critical
